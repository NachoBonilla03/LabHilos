La versión secuencial procesa todos los datos utilizando un solo hilo, recorriendo el arreglo completo de números uno por uno para calcular las funciones principales del programa (suma, el promedio, el mínimo y el máximo). 
Esta solución es la más simple y fácil de entender, pero el rendimiento depende totalmente de un único "trabajador" ejecutando el código, por lo que puede tardar más cuando se ejecuta el código con grandes cantidades de datos. 

En cambio, la versión paralela divide el arreglo en varias partes y asigna cada sección a diferentes hilos de ejecución que trabajan al mismo tiempo. Para realizar esto, se utiliza la clase Worker, que se encarga de procesar una parte específica del arreglo. Cada hilo ejecuta un objeto Worker, el cual calcula resultados parciales como la suma, el mínimo, el máximo y la cantidad de números en su segmento del arreglo. Al finalizar la ejecución de todos los hilos, estos resultados parciales se combinan para obtener el resultado total. Tomando esta solución, el procesamiento puede utilizar múltiples hilos para reducir el tiempo de ejecución. Sin embargo, esta solución introduce una mayor complejidad en el código debido a la creación, coordinación y sincronización de los hilos.
